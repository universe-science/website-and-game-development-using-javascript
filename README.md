# 📋 Пошаговая инструкция

Далее ты найдёшь все необходимые инструкции, чтобы подготовить свой компьютер к занятиям, а также создать, запустить и увидеть в браузере свой первый сайт.

## 1. Регистрация на GitHub (даже если тебе меньше 13 лет)

GitHub — это сайт, где мы будем хранить код. Чтобы создать аккаунт, нужно немного помочь родителей.

#### Если тебе уже есть 13 лет
1. Перейди на [github.com](https://github.com) и нажми **"Sign up"**.
2. Придумай логин (например, имя и фамилию латиницей), введи свою почту и пароль.
3. Подтверди почту — придёт письмо с кодом.

#### Если тебе меньше 13 лет
Правила GitHub требуют согласия родителей. Сделайте так:
1. Попроси маму или папу помочь создать аккаунт **на их почту**.
2. При регистрации укажи дату рождения (свою настоящую) — GitHub может попросить parental consent.
3. Родитель должен будет дать разрешение (обычно через подтверждение по почте или специальную форму). Мы можем сделать это вместе на уроке.

После регистрации обязательно **запомни логин и пароль**! Они понадобятся для входа в VS Code позже.



## 2. Установка VS Code

VS Code — это редактор кода, в котором мы будем работать.

#### Шаг 1. Скачай установщик
1. Перейди на [code.visualstudio.com](https://code.visualstudio.com/)
2. Нажми большую синюю кнопку **"Download for Windows"** (или для своей ОС).

#### Шаг 2. Установи с важной галочкой
1. Запусти скачанный файл.
2. Нажимай **"Далее"**, пока не дойдёшь до экрана **"Выберите дополнительные задачи"**.
3. **Обязательно отметь галочкой** пункт **"Добавить действие PATH (переменные среды)"** (или "Add to PATH").
4. Нажми **"Установить"**, потом **"Готово"**.
5. VS Code откроется автоматически.



## 3. Установка Deno (ручной способ)

Мы будем использовать Deno для написания серверной части и не только. Этот способ работает без проблем в России.

#### Шаг 1. Скачай архив с GitHub
1. Открой страницу [https://github.com/denoland/deno/releases](https://github.com/denoland/deno/releases)
2. Найди самый свежий релиз (первый сверху).
3. В разделе **Assets** скачай файл **`deno-x86_64-pc-windows-msvc.zip`** (для Windows 64-бит). Если у тебя другая ОС, скажи учителю — выберем нужный файл.

#### Шаг 2. Распакуй и положи в нужную папку
1. Распакуй архив — внутри будет файл `deno.exe`.
2. Создай на диске `C:` папку `C:\deno` и перемести туда `deno.exe`.

#### Шаг 3. Добавь Deno в PATH
1. Нажми `Win + R`, введи `sysdm.cpl` и нажми Enter.
2. Вкладка **"Дополнительно"** → кнопка **"Переменные среды"**.
3. В нижнем списке (Системные переменные) найди `Path`, выдели и нажми **"Изменить"**.
4. Нажми **"Создать"** и введи `C:\deno`. Нажми **ОК** во всех окнах.
5. **Если VS Code уже открыт, перезапусти его**, чтобы изменения вступили в силу.

#### Проверка
Открой терминал (в VS Code нажми `` Ctrl + ` `` ) и введи:
```bash
deno --version
```
Должна появиться надпись вроде `deno 1.46.3` — значит всё хорошо.



## 4. Создаём рабочую папку

На рабочем столе создай папку с именем **Projects**.  
Внутри неё создай ещё одну папку — **helloworld**.



## 5. Открываем папку в VS Code

1. Запусти VS Code (если ещё не запущен).
2. Нажми **"File"** → **"Open Folder..."** (или **"Открыть папку"**).
3. Найди на рабочем столе папку **Projects**, зайди в неё, выбери папку **helloworld** и нажми **"Выбрать папку"**.
4. Если VS Code спросит, доверяете ли вы авторам, нажми **"Yes"**.

Теперь VS Code показывает содержимое папки helloworld (пока там пусто).



## 6. Установка расширения Live Share

Live Share позволяет работать над кодом вместе — учитель сможет подключаться к твоему проекту и помогать.

1. В VS Code открой вкладку **Расширения** (иконка квадратиков слева или `Ctrl+Shift+X`).
2. В поиске введи **`Live Share`**.
3. Найди расширение от **Microsoft** (иконка с тремя звеньями цепи) и нажми **"Установить"**.
4. После установки в левом нижнем углу появится кнопка **"Live Share"**. Нажми её и войди в свой **GitHub аккаунт** (тот, что создали в шаге 1). Подтверди вход в браузере.
5. В левом нижнем углу теперь будет гореть твоя аватарка — значит, Live Share готов к работе.



## 7. Запускаем сессию Live Share и даём доступ учителю

1. Нажми кнопку **"Live Share"** в левом нижнем углу VS Code.
2. Через пару секунд появится всплывающее окошко со ссылкой. Скопируй её (нажми **"Copy Link"**).
3. Отправь эту ссылку учителю (например, в чат).
4. Учитель перейдёт по ссылке, и вы будете работать в одном проекте. Он сможет видеть твой код и помогать.



## 8. Пишем первый сервер (Hello World)

Создадим файл `server.ts` в папке helloworld.

1. В левой панели VS Code (проводник) нажми правой кнопкой мыши и выбери **"New File"**. Назови его `server.ts`.
2. Открой этот файл и вставь следующий код:

```typescript
// Простой веб-сервер на Deno
Deno.serve((req) => {
  return new Response("Привет, мир! 🦕", {
    headers: { "content-type": "text/plain; charset=utf-8" },
  });
});

console.log("Сервер запущен на http://localhost:8000");
```

3. Сохрани файл (`Ctrl+S`).

Теперь запустим сервер. Открой терминал в VS Code (`` Ctrl + ` `` ) и выполни команду:
```bash
deno run --allow-net server.ts
```
Ты увидишь сообщение: `Сервер запущен на http://localhost:8000`.  
Открой браузер и перейди по адресу [http://localhost:8000](http://localhost:8000) — там должно быть написано «Привет, мир!».



## 9. Пробрасываем порт и открываем публичный доступ

Сейчас сервер работает только на твоём компьютере. Чтобы учитель (или кто угодно) мог открыть твой сайт через интернет, нужно создать туннель.

1. В VS Code открой вкладку **"Порты"** (Ports).  
   Она находится в нижней панели, рядом с «Терминал» и «Проблемы». Если не видно, нажми на три точки и выбери **"Ports"**.
2. Ты увидишь список портов. Скорее всего, порт `8000` уже отобразился автоматически. Если нет, нажми **"Forward a Port"** (или "Добавить порт") и введи `8000`.
3. В строке порта нажми правой кнопкой мыши и выбери **"Port Visibility"** → **"Public"** (или **"Открытый доступ"**).  
   После этого рядом с портом появится публичный URL (например, `https://...`).
4. Нажми на этот URL (или скопируй и открой в браузере). Теперь твой сайт доступен из любой точки мира!

**Важно:** Этот URL работает, пока запущен VS Code и сервер. Если закроешь VS Code, туннель закроется.
